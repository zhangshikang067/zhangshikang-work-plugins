# OKR 参考文件模板

这是一个示例文件，展示 skill 期望的 OKR 文件格式。skill 运行时会自动查找 `references/{git用户名}-okr.md`（例如 `references/张三-okr.md`）。如果不存在，会通过交互提示用户提供。

## 文件格式要求

OKR 文件应包含以下信息，方便 skill 将工作事项归类到正确的 KR：

- **Objective 和 Key Result 列表**：每个 KR 的标题、权重和目标描述
- **适合归入该 KR 的事项关键词**：帮助自动归类
- **归类原则**：如何处理跨 KR 事项、无法归类的事项

## 示例结构

```markdown
## O1：[Objective 标题]

### KR1：[Key Result 标题]【权重】

目标：[具体目标和衡量标准]
衡量方式：[如何衡量]
信心指数：[0-100%]

适合归入该 KR 的事项：
- [关键词1]、[关键词2]、[关键词3]
- [关键词4]、[关键词5]

### KR2：...
```

## 使用原则

- 优先把工作事项归到最贴近的 KR；无法确定时，说明"可能归属"，不要强行匹配
- 如果事项属于多个 KR，优先选择本周主要贡献最大的 KR，必要时在 bullet 中体现次要关联
- 不要编造 OKR 进度百分比、事故等级、准确率或提效比例；只有用户提供数据时才写具体数字
- 无法明确归属的事项放到"其他工作"分组

## 快速归类建议（可选）

可以在文件末尾加一个快速归类映射表，列出常见关键词到 KR 的对应关系：

```markdown
- 关键词A、关键词B：优先归入 `O1-KR1`
- 关键词C、关键词D：优先归入 `O1-KR2`
```
