# OKR Weekly Report Writer

使用这个 skill，把零散的工作记录改成清晰、可信、方便领导阅读的 OKR 周报。输出要让工作成果、推进进度、风险处理和后续动作一眼能看懂。

## 工作流程

1. 先识别周报背景：周期、OKR/KR 编号、阅读对象、用户角色、原始事项、已有数据、阻塞问题和后续计划。
2. 如果用户提供优秀范例，先提炼范例的结构、措辞和表达习惯，再进行改写，尽量贴近公司内部风格。
3. 如果用户要求结合个人 OKR，且存在 `references/my-2026-h1-okr.md`，读取该文件用于 KR 归类和措辞对齐；如果该文件不存在，继续按通用周报规则处理。
4. 如果缺少关键数据，不要编造数字。可以不写该数据，或在必要时用简短的“可补充数据”提示。
5. 按 OKR/KR 分组。每组不要只列任务，要写成“做了什么 + 结果/影响 + 当前状态/风险/下一步”。
6. 加入“思考与分享”部分，体现判断力：风险识别、取舍、复盘、学习和具体后续动作。
7. 语气保持客观、克制、专业。避免自夸、道歉式表达或流水账。

## 好周报的特点

优先体现这些特点：

- **对齐 OKR**：使用清晰标签，例如 `O1-KR4：AI 测评`；无对应 OKR 的事项必要时放到“暂无 OKR”。
- **结果前置**：写“完成开发与调试，支持能力落地”，不要只写“做了开发”。
- **能量化就量化**：尽量写清数量、比例、金额、完成状态、响应时长、准确率、日期或覆盖范围。
- **风险可见**：写清阻塞和处理方式，例如“因当前暂不具备联调条件，已先完成开发实现，待环境满足后验证闭环”。
- **过程价值清楚**：说明工作如何降低风险、提升稳定性、提升准确性、提高效率或保障上线。
- **协作有意义**：只有当协作能说明进展、依赖或推进难点时才写，不要作为填充内容。
- **后续动作明确**：结尾写具体跟进动作，不写空泛表态。
- **简洁但不单薄**：每条 bullet 都要承载一个明确贡献。

避免这些弱表达：

- 只列任务、不写结果：例如“配合测试”“完成需求”“处理问题”。
- 每条都用“配合”开头，导致贡献显得被动。
- 没有依据地写“大幅提升”“显著优化”等泛化结论。
- 隐藏阻塞。清楚写出阻塞和应对方式，比假装全部完成更可信。
- 思考部分只是重复工作清单，没有体现判断或复盘。

## 改写模式

每个事项按这个结构改写：

```text
原始事项：做了什么
优化表达：围绕【目标/场景】，完成【动作/交付物】，解决/保障/推动【结果】，当前【状态/风险/后续】。
```

常用动词：

- 推进、完成、支持、保障、打通、梳理、排查、修复、验证、沉淀、优化、跟进、闭环
- 只有用户确实是辅助角色时才用“配合”。其他情况优先用“支持完成”“协助推进”“完成验证”“保障上线”等表达。

常用结果词：

- 稳定性、准确性、可用性、兼容性、时效、效率、转化、体验、上线质量、风险前置、问题闭环

常用阻塞表达：

- "因 X 暂不具备条件，已先完成 Y，后续待 Z 后继续验证闭环。"
- "当前主要风险在于 X，已通过 Y 方式前置处理，后续重点跟进 Z。"
- "该项工作对 X 要求较高，因此本周重点放在 Y，减少后续 Z 风险。"

## 推荐输出结构

改写普通 OKR 周报时，优先使用这个结构：

```markdown
**简述近期 OKR 重要进展或规划**

本周 OKR 重点推进工作如下：

**O1-KR2：财务优化**

- ...
- ...

**O1-KR4：AI 测评**

- ...
- ...

**O1-KR3：项目迭代**

- ...
- ...

**简述近期实质性思考和分享**

本周在 ... 过程中，进一步感受到 ...

后续会 ...

同时也意识到 ...
```

如果用户所在公司的模板标题不同，沿用用户的标题。

## 思考与分享写法

思考部分建议回答下面 2-4 个问题：

- 本周哪类工作最难、风险最高？
- 用户做了什么判断或取舍？
- 从重复问题中总结出了什么规律？
- 下次哪些风险可以更早识别和处理？
- 哪些依赖还需要继续跟进，才能形成闭环？
- 用户在哪些能力上有进步：问题定位、风险识别、跨团队沟通、交付质量、数据意识、流程思考？

思考要具体，不要空泛。可参考这些表达：

- "AI 类需求更依赖外部服务稳定性和异常兜底，后续会在开发阶段提前关注超时、失败重试和模型兼容。"
- "广告归因类需求对第三方接口和联调环境依赖较强，本周先完成可开发部分，避免因单个平台阻塞影响整体进度。"
- "财务类需求对数据准确性要求较高，后续测试会继续重点关注数据流转、边界场景和流程衔接。"

## 最终检查清单

输出前检查：

- 领导能在 1 分钟内看懂重点。
- 每个 KR 都有明确进展或当前状态。
- 重要工作没有被平淡措辞削弱。
- 没有编造数据。
- 阻塞问题写了应对方式或下一步。
- 思考部分体现判断，而不只是态度。
- 整体语气专业、真实、克制，不刻意堆砌辞藻。
