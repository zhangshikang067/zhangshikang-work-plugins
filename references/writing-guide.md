# 月度绩效书写指南

## 核心原则

月度绩效自评的目标：让上级看到你的**高产出、深思考、强自驱**。

- 工作快报要**扎实**：用事实和数字说话，体现业务价值
- 经验要**有深度**：从具体场景提炼方法论，体现成长
- 不足要**有策略**：包装成进取心和改进意愿，不是真的暴露短板

---

## 主要工作快报

### 写法要点

每条工作要体现三要素：**动作 + 结果 + 价值**

| 维度 | 示例 |
|------|------|
| 动作 | 完成 XX 系统全链路开发 |
| 结果 | 接口响应时间降低 60% |
| 价值 | 支撑业务方 YY 场景上线，预计覆盖 N 万用户 |

### 量化参考

从周报中提取可用数据，没有精确数字时用范围描述：
- 数量级："完成 10+ 个需求"、"修复多个线上缺陷"
- 覆盖面："覆盖全部核心场景"、"支持 3 个业务方"
- 对比提升："较之前有明显提升"、"性能指标达到预期目标"

### 好与不好的对比

```
好：完成支付模块全链路开发与联调，修复 3 个阻塞性问题，
    按期完成里程碑验收，支撑业务侧按时上线

不好：做了支付模块的开发和联调
```

## 经验写法（深度分析）

经验是展示思考深度的关键部分，不能写成空泛的总结。

### 经验提炼框架

从周报的工作内容中，思考以下问题来提炼经验：

1. 这个月遇到的**最难的问题**是什么？怎么解决的？
2. 做了什么**关键判断或取舍**？为什么这样选？
3. 从重复出现的问题中总结出了什么**规律或方法**？
4. 在哪些**能力维度上有明显提升**？
5. 哪些经验是**可以复用到后续工作**中的？

### 经验呈现结构

```text
场景（触发点）→ 认知（发现了什么）→ 方法（怎么做的/怎么做更好）→ 沉淀（后续怎么复用）
```

### 示例

```text
在处理 XX 系统高并发场景时，发现原有架构在峰值场景下存在性能瓶颈。
通过分析链路耗时，定位到 YY 环节是主要卡点，采用 ZZ 方案优化后，
峰值处理能力提升 3 倍。这次经历让我更深刻地认识到：
在架构设计中，提前识别热点路径比事后优化更高效。
已将这一思路整理成架构评审 checklist，供团队后续参考。
```

### 经验方向参考

根据实际工作内容选择最贴合的方向：
- **技术判断力**：方案选型时的决策过程和理由
- **问题定位能力**：复杂问题的排查思路和方法
- **推动力**：跨团队/跨角色推动事情落地的经验
- **流程改进**：发现流程问题并主动优化的案例
- **风险意识**：提前识别风险并前置处理的实例
- **能力跃迁**：从执行某个任务到独立设计某个方案的成长

## 不足写法（积极美化）

不足的写法核心：让领导觉得你有**高标准、有自省力、已经在改进**。

### 美化策略一：高标准视角

把"做得不好"转化为"追求更高标准时的发现"：

| 原始表达 | 美化后 |
|---------|--------|
| 代码质量不够好 | 在追求更高交付质量的过程中，发现代码审查环节还有进一步精细化的空间 |
| 进度有延迟 | 在保证质量的前提下，发现项目管理颗粒度还可以更细，以提升交付的确定性 |
| 沟通不够及时 | 随着协作范围扩大，意识到在信息同步的及时性和透明度上还有提升空间 |

### 美化策略二：主动发现型

把问题包装为你主动发现并正在解决的改进项：

```text
在复盘本月项目交付过程时，主动梳理了 XX 环节的效率瓶颈，
发现通过引入 YY 机制可以进一步提升。已开始调研相关方案，
计划下月在 ZZ 项目中试点验证。
```

### 美化策略三：成长阶段型

把不足定位为能力成长过程中的自然阶段：

```text
随着本月承担的职责从 XX 拓展到 YY，对 ZZ 能力提出了更高要求。
目前在 WW 方面已开始系统学习，预计下月能在实际项目中实践验证。
```

### 不足的固定结构

每条不足都包含三个要素：
1. **场景/背景**：在什么情况下发现的
2. **改进方向**：不是问题本身，而是要提升的方向
3. **已有行动**：已经做了什么或计划做什么

```
不足示例（完整版）：
在推进多个并行项目的过程中，发现时间分配的精细化程度还有提升空间。
已开始使用优先级矩阵进行任务管理，月末回顾时效率已有改善，
下月计划进一步优化并结合团队工具实现可视化追踪。
```

### 不足的禁忌

- 不要写"还需要继续努力"这类空话——必须有具体改进动作
- 不要写真正的能力缺陷或重大失误——那是减分项
- 不要写太多条不足——2-3 条即可，多了显得问题多
- 不要写和别人对比的不足——只谈自己的提升空间

## 落实情况

每条上月建议需要闭环回应，展现执行力和责任心。

### 已完成

```text
1. 建议提升代码审查质量：已完成。引入 checklist 机制，
   本月审查覆盖率达 100%，线上缺陷数较上月下降。
```

### 推进中

```text
2. 建议完善技术文档：大部分完成。核心模块文档已补齐（4/5），
   剩余部分已排入下月计划，预计中旬完成。
```

### 未完成（美化原因）

```text
3. 建议搭建自动化测试框架：因本月优先支撑 XX 紧急上线任务，
   暂缓推进。技术方案已完成预研，下月正式启动。
```
