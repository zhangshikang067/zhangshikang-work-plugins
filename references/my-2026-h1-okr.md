# 个人 2026 H1 OKR 参考

使用场景：当用户要求“结合我的 OKR 写周报”“按我的 2026 H1 OKR 对齐周报”“帮我判断事项归属哪个 KR”时，读取本文件。不要把本文件内容当作通用周报规则；分享 skill 给他人时，可以删除整个 `references/` 目录，保留通用写作能力。

## 使用原则

- 优先把工作事项归到最贴近的 KR；无法确定时，说明“可能归属”，不要强行匹配。
- 写周报时突出该事项对 KR 的贡献：稳定性、数据准确率、上线质量、业务提效、安全合规、AI 工具应用或架构沉淀。
- 如果事项属于多个 KR，优先选择本周主要贡献最大的 KR，必要时在 bullet 中体现次要关联。
- 不要编造 OKR 进度百分比、事故等级、准确率或提效比例；只有用户提供数据时才写具体数字。

## O1：深度支持业务项目创新迭代，全面助力拉新增长及业务提效

### KR1：拉新数据看板支持【30%】@Siyou Bao

目标：全力配合拉新数据看板建设，确保覆盖全链路数据。2 月底上线，保障数据准确率 >99.9%，上线首月无 P0 级重大生产事故。

衡量方式：时间或数据衡量。

信心指数：80%。

适合归入该 KR 的事项：

- 拉新数据看板建设、数据链路、数据口径、数据校验。
- 看板上线支持、数据准确率保障、生产事故风险控制。
- 与拉新增长相关的数据支持、埋点、统计、报表、监控。

### KR2：财务系统保障【30%】@Siyou Bao

目标：持续保障财务系统核心链路稳定性，支持后续功能迭代。财务系统订单数据链路可用性达到 99.99%，订单与财务数据日对账差异数为 0，季度内无 P1 级以上财务数据事故。

衡量方式：时间或数据衡量。

信心指数：85%。

适合归入该 KR 的事项：

- 财务系统需求、财务优化、财务测试、订单财务链路。
- 日对账、数据一致性、订单数据准确性、异常数据排查。
- 财务系统稳定性、可用性、核心链路保障、财务事故风险前置。

### KR3：项目迭代优化【30%】@Siyou Bao

目标：高质量支持项目迭代，引入 AI 代码评审和检查（Pass 率 100%），确保负责的项目上线无回滚、无 P0 级故障。

衡量方式：时间或数据衡量。

信心指数：80%。

适合归入该 KR 的事项：

- 普通业务项目开发、项目迭代、官网展示、广告归因、需求上线。
- 代码质量、AI 代码评审、上线验证、回滚风险控制、P0 故障风险控制。
- 需求排期、开发实现、联调、上线支持、发布后验证。

### KR4：AI 产品支持【10%】@Siyou Bao

目标：支持 AI 功能迭代，完善模型 token 消耗可观测机制，提高业务系统使用大模型的易用性、可观测、可维护性，并支撑新增业务量落地。

适合归入该 KR 的事项：

- 百度 OCR、豆包系模型、大模型 API 接入与调试。
